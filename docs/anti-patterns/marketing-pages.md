# Marketing Page Anti-Patterns

Marketing pages fail when they become a taxonomy diagram or a pitch deck.

## Homepage Anti-Patterns

Do not show:

- every product
- every repo
- every agent
- every customer segment
- every future idea
- a fake "loop" with five equal cards
- a bottom CTA by default
- a "proof" section without real proof
- an OSS grid just because repos exist

Homepage should answer:

- What is this?
- Why does it matter now?
- What is the one product spine?
- What can I inspect?

For Tangle-style pages, the spine is likely:

- Sandbox
- Router
- Intelligence

Everything else is secondary unless the page is specifically about it.

## Section Anti-Patterns

Bad section structure:

1. eyebrow label
2. heading
3. paragraph
4. card grid
5. CTA

This shape is almost always AI-generated marketing sludge.

Better:

- one strong heading
- one visual product artifact
- one sentence if necessary
- no labels unless they are real data

## Product Page Anti-Patterns

Do not:

- describe the product with abstractions before showing it
- use screenshots as decorative thumbnails
- create invented UI mockups when a real UI exists
- hide the actual product behind icons
- use cards to avoid making a diagram
- make the first viewport a split text/card layout unless the card is the product itself

Show:

- real screenshot
- real terminal output
- real trace
- real model route
- real eval result
- real artifact

## Technical Product Marketing Anti-Patterns

Avoid:

- "enterprise-grade" with no enterprise constraint
- "secure" with no threat model
- "observable" with no trace example
- "evaluated" with no rubric or heldout set
- "open source" with no repo, install path, or live maintenance signal
- "developer-first" with no command or API

## Navigation Anti-Patterns

Do not expose a bad information architecture in the nav.

If pages are weak, remove the links. A small site with Blog and Research is better than a large broken site.

Nav should not be a company org chart.
