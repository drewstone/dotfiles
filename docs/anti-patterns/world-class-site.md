# World-Class Site Doctrine

This file defines what "world-class site" means for a serious technical startup.

## What World-Class Does Not Mean

It does not mean:

- more pages
- more CTAs
- more sections
- more product labels
- more gradients
- more cards
- bigger typography everywhere
- a generic "Apple/Airbnb/OpenAI" imitation
- one page that explains the entire company

## What World-Class Means

A world-class site:

- knows what it is selling
- knows what it is not showing
- has a product point of view
- uses real artifacts
- has a recognizable visual system
- has fewer, stronger pages
- makes the next decision obvious
- preserves trust by not overstating claims

## Tangle Brand Thesis

Tangle is serious infrastructure for agents that improve from evidence.

The point of view:

> Agents only improve when the run is observable, reproducible, scored, and promoted from evidence.

Primary product spine:

- Sandbox: where the agent runs.
- Router: how the model call is controlled.
- Intelligence: how traces become diagnosis, evals, and improvement.

Secondary developer surfaces:

- agent-runtime
- agent-eval
- agent-app
- agent-sdk

Examples and applications:

- browser-agent
- blueprint-agent
- tax-agent
- legal-agent
- gtm-agent

Anti-pattern:

Do not show all three layers at once on the homepage. That is an org chart, not a site.

## Site Shape

Keep:

- Home
- Blog
- Research

Hide or delete until they have proof:

- product pages
- agent pages
- protocol pages
- ecosystem pages
- brand pages
- status/security/legal pages unless operationally required

## Homepage Doctrine

The homepage should do one thing:

Make the product spine feel inevitable.

Allowed:

- hero with strong visual identity
- one product artifact that shows Sandbox, Router, and Intelligence through one run
- latest research/blog surfaced cleanly
- one docs/workbench action if useful

Banned:

- product catalog
- repo catalog
- agent catalog
- five-card loop
- "how it works" steps
- bottom CTA by default
- tiny section labels
- traffic-light trace table
- cards that say Sandbox / Router / Agent Eval / Browser Agent

## Research Doctrine

Research should be:

- light theme
- calm
- readable
- evidence-led
- sparse
- dated
- willing to be incomplete

Research should not be:

- dark marketing
- SEO comparison pages
- product CTAs
- fake tracks
- labeled system layers
- "Research" as both label and title

## Blog Doctrine

Blog can carry:

- technical explainers
- comparisons
- implementation notes
- launch notes
- essays

Blog should not pretend to be research unless it has method, evidence, and a falsifiable claim.

## Product Page Doctrine

No product page ships without at least one real artifact:

- screenshot
- trace
- model route
- eval output
- terminal output
- architecture diagram from actual system boundaries
- demo video

If the artifact does not exist, the page should not exist.

## Visual Identity Guardrails

Tangle can use:

- dark purple atmospheric hero
- light editorial research/blog
- technical diagrams
- real screenshots
- trace-derived visuals
- restrained animation

Tangle should avoid:

- green production palette unless explicitly chosen
- decorative gradient blobs
- one-note purple everywhere
- Chrome-heavy fake UI
- card farms
- tiny labels
- fake terminal windows as decoration
