# Principles

## The Enemy

The enemy is not minimalism. The enemy is fake structure.

Fake structure is what agents add when they do not understand the product, audience, or artifact:

- tiny labels above headings
- card grids with repeated copy
- taxonomy chips
- numbered loops
- labels like `observe`, `sandbox`, `router`, `agent eval`, `blueprints`
- CTA strips that exist because the template wants a CTA
- research pages with non-research comparison posts
- marketing pages that explain instead of show

## What Good Looks Like

Good work has fewer visible parts and more force.

- One viewport makes one decision clear.
- A heading carries the idea without an eyebrow label.
- A product page shows the product directly.
- Research argues from evidence, not category names.
- Blog writing teaches something specific.
- UI copy names actions, state, values, and consequences.
- Visual design uses artifacts, screenshots, diagrams, media, motion, or data.

## Anti-Patterns By Class

### Label Addiction

Symptoms:

- uppercase micro-label above every heading
- small colored label to make a boring section feel designed
- card labels that repeat the card title
- labels for internal taxonomy instead of user meaning

Kill test:

Delete the label. If the section loses meaning, rewrite the heading or delete the section.

### Taxonomy Worship

Symptoms:

- products shown as peer chips because they exist internally
- repo names on marketing pages before the user understands the system
- `Sandbox`, `Router`, `Browser Agent`, `Agent Eval`, `Blueprints` presented as a row with equal weight
- internal layer names used as visual hierarchy

Kill test:

Ask what the user is deciding. If the answer is not "choose between these taxonomy items," do not show them as options.

### Template Gravity

Symptoms:

- hero, feature grid, proof section, OSS grid, CTA section, footer
- same card component repeated with different labels
- bottom CTA because landing pages usually have one
- "How it works" section that restates the product in steps

Kill test:

Remove the section. If no user decision gets worse, it was template residue.

### Text As Design

Symptoms:

- long explanatory sections with no artifact
- paragraphs trying to compensate for missing product proof
- code snippets, terminal boxes, or trace tables used as decoration
- labels and captions where screenshots, diagrams, or real controls should be

Kill test:

Replace the text with a screenshot, diagram, visual state, data table, or product artifact. If no artifact exists, say the product proof is missing.

### One-Viewport Overload

Symptoms:

- all products, repos, agents, and future ideas on one page
- no primary hierarchy
- every noun becomes a card
- the user sees a company map instead of a reason to care

Kill test:

Pick one: product spine, audience path, proof artifact, or argument. Hide the rest.
